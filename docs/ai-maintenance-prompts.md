# AI Maintenance Prompts

Use these prompt files whenever the project has changed enough that future development context should be refreshed. The primary goal is to keep the repository easy for an AI coding assistant or a human developer to understand, extend, verify, and safely continue.

## Prompt Files

- [Refresh base development docs](ai-maintenance/refresh-base-docs.md)
- [Run an AI maintainability refactor](ai-maintenance/refactor.md)
- [Sync hosted app with local](ai-maintenance/sync-hosted-with-local.md)
- [Commit and push a context checkpoint](ai-maintenance/commit-and-push.md)
- [Run the full AI maintenance cycle](ai-maintenance/full-cycle.md)

## How To Cite A Prompt

Reference the specific file for the workflow you want:

```text
Please use docs/ai-maintenance/refactor.md.
```

```text
Please use docs/ai-maintenance/refresh-base-docs.md, then docs/ai-maintenance/commit-and-push.md.
```

```text
Please use docs/ai-maintenance/sync-hosted-with-local.md.
```

```text
Please use docs/ai-maintenance/full-cycle.md.
```

This index remains as the discovery page. The individual files are the canonical prompts.
